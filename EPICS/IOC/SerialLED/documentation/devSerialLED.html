Using SerialLED instrument support in an application

Several files need minor modifications to use SerialLED instrument support in an application.

  1. Add the full path to the SerialLED support directory to the application configure/RELEASE file:
    SerialLED=xxxx/modules/instrument/serialled/<release>
    Where <release> is the release number of of the SerialLED support.
  2. Add instrument support to application database definition file
    The application database definition file must include the database definition files for the SerialLED instrument and for any needed ASYN drivers. There are two ways that this can be done:
    • If you are building your application database definition file from an xxxInclude.dbd file you include the additional database definitions in that file:
      include "base.dbd"
      include "devSerialLED.dbd"
      include "drvAsynIPPort.dbd"
    • If you are building your application database definition file from the application Makefile you specify the aditional database definitions there:
      xxx_DBD += base.dbd
      xxx_DBD += devSerialLED.dbd
      xxx_DBD += drvAsynIPPort.dbd
  3. Add the SerialLED support libraries to the application
    You must link the SerialLED support library and the ASYN support library with the application. Add the following lines:
    xxx_LIBS += devSerialLED
    xxx_LIBS += asyn
    before the
    xxx_LIBS += $(EPICS_BASE_IOC_LIBS)
    in the application Makefile.
  4. Load the SerialLED support database records in the application startup script:
    cd $(SerialLED)      (cd SerialLED if using the vxWorks shell)
    dbLoadRecords("db/devSerialLED.db,"P=<P>,R=<R>,L=<L>,A=<A>")
    You'll have to provide appropriate values for the PV name prefixes (<P> and <R>), the link number (<L>) and the device address (<A>). The link number must match the value specified in an ASYN drvxxxxxConfigure command.

Installation and Building

After obtaining a copy of the distribution, it must be installed and built for use at your site.
  1. Create an installation directory for the module. The path name of this directory should end with modules/instrument/serialled.
  2. Place the distribution file into this directory.
  3. Execute the following commands:
    cd modules/instrument/serialled
    gunzip serialled<release>.tar.gz
    tar xvf serialled<release>.tar
    cd <release>
    Where <release> is the release number of of the SerialLED support.
  4. Edit the configure/RELEASE file and set the paths to your installation of EPICS base and ASYN.
  5. Execute make in the top level directory.
